## 二、特别审计

### 2.1 Golden Set 是否过拟合

**风险等级：中**

- Golden Set 的 `expected_items` 已校准为 RuleOnly 实际能提取的值（而非理想值），"100% 通过率"本质上是**边界对齐率**而非**语义覆盖率**
- 150 条样本中 30 条依赖 LLM（llm_expected_items），119 条纯规则可覆盖
- 人工构造的样本缺乏真实群聊的自然噪声（错别字、半截句子、混合话题）
- **不会影响 Demo**，因为 Demo 场景可控。**会影响真实可用性**，真实群聊中的消息变化远超 150 条样本能覆盖的范围

**缓解**：Golden Set 标注过程文档化了每条的语义理由，不是盲目匹配 LLM 输出。但应被视为"已知覆盖边界"而非"完美系统"。

### 2.2 隐式语义提取是否仍不稳定

**风险等级：中**

- 3 条非确定性失败（GS-031/GS-116/GS-119）来自 LLM temperature=0.1 的波动
- 同一消息在不同 run 中可能得到不同类型（如 GS-041 有时给 decision 有时给 next_step）
- 30 条 LLM 案例中 ~10% 存在 run-to-run 差异
- **不会影响 Demo**（单次 run 效果可接受）。**会小幅影响真实可用性**（同一消息重复处理可能得到不同结果）

**缓解**：`upsert_items` 的四层去重会合并重复 key，降低不一致的影响。将 temperature 降到 0 可消除但会降低创造性识别。

### 2.3 真实飞书复杂消息类型覆盖不足

**风险等级：高**

- 当前 `_extract_text()` 和 `demo_sync_messages.py` 主要处理 `msg_type: "text"` 消息
- **未覆盖的消息类型**：
  - `post`（富文本）：content 是 JSON 数组，可能含 @提及、图片、链接。`_extract_text` 有 post 解析逻辑但**未用真实 post 消息验证**
  - `image`/`file`/`audio`/`video`：content 为空或不含文本，当前 `_event_text` 会返回空字符串跳过
  - `share_chat`/`share_user`：转发消息，含嵌套内容
  - `interactive`（卡片消息）：按钮、表单等，文本提取可能失败
- **Demo 风险**：如果评委群中有 post 或 图片消息，系统会跳过它们（不会崩溃），但会丢失这些消息中的协作信息
- **真实风险**：飞书群聊中 post 消息（含 @的富文本）占比可达 30-50%

### 2.4 权限隔离只是软隔离

**风险等级：低（Demo）/ 高（生产）**

- 基于 `project_id` 参数做数据隔离，无飞书 open_id 校验
- 任何人改 `--project-id` 参数即可访问其他项目记忆
- **不影响 Demo**（本地单用户场景）。**不建议现阶段修**（属 P2 生产化需求）

### 2.5 Decision/Deadline/Member_status 更新逻辑误覆盖风险

**风险等级：中**

- **Decision**：Layer 4 跨 key 覆盖使用共享关键词检测。风险：两个不同主题但碰巧共享关键词的 decision 可能被误覆盖（如 "用 React 做前端" vs "用 React Native 做移动端"）。不过阈值设置较保守（需同时满足覆盖信号词+共享词），实际误覆盖概率低
- **Deadline**：多个 deadline 因 `_hash_key(text)` 生成不同 key，不会自动关联。如果两次讨论相同 deadline（如"改成下周一"→"再改成下周五"），两条独立存在，**不会自动 supersede** — 这是设计局限，不是 bug
- **Member_status**：`_hash_key(text)` 生成不同 key，同一人的不同状态表达不会自动关联。"请假"→"休假回来"会作为两条独立 member_status 共存。如果消息包含了关键词但语义相反（如"不请假"），会被 RuleBased 提取为成员状态
- **不会影响 Demo**。**真实场景需要关注**：同一 topic 多次变更可能在交接摘要中呈现重复/冲突信息

## 四、审计结论

**当前状态可 Demo**。四个优先级的修复显著提升了系统的可信度和完整性。主要遗留风险集中在真实消息类型覆盖（P0-1/P0-2）和 LLM 稳定性（P1-1），均可在 2 小时内解决。

Golden Set 150/150 的 RuleOnly 通过率应被理解为"已知覆盖边界的精确测量"而非"完美语义理解"。Hybrid 98.0% 证明了 LLM 对隐式语义的有效补充，但仍有 3 条非确定性边界案例。
