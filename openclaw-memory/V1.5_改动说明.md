# OpenClaw Memory Engine 版本改动说明

本文档汇总 V1.1 至 V1.8 的所有版本改动。

---

## V1.1 — 可信提取链路（基础）

**日期**：2026-04-25

**核心改进**：
- 新增 `LLMExtractor`，将 raw events 转换为结构化 `MemoryCandidate`
- Schema 校验 + 证据锚点验证（source_refs 必须非空、message_id 必须在输入中）
- LLM 失败时回退到 `RuleBasedExtractor`

**新增文件**：`candidate.py`、`llm_provider.py`、`handoff.py`、`action_planner.py`

---

## V1.5 — P0 核心能力

**日期**：2026-04-26

| 改动 | 文件 | 功能 | 借鉴来源 |
|------|------|------|----------|
| Prompt Grounding | `extractor.py` | 代词/时间/空间解析规则 | agent-memory-server |
| 三层去重 | `store.py` | ID + Hash + Semantic 去重 | agent-memory-server |
| Debounce 提取 | `engine.py` | coalescing debounce | agent-memory-server |

### extractor.py
- **新增 `_extract_mentions()`**：从飞书消息 `at_list` 字段提取 @提及映射
- **修改 `_build_prompt()`**：Prompt 增加上下文时间、参与人员、代词/时间/空间解析规则、输出格式说明

### store.py
- **新增 `_compute_text_similarity()`**：字符 bigram Jaccard 相似度
- **修改 `upsert_items()`**：三层去重（Identity Key → Content Hash → Semantic Similarity）

### engine.py
- **修改 `MemoryEngine`**：新增 `debounce_seconds` 和 `_last_process_time`，coalescing debounce

## V1.6 — 可靠性修复 + Golden Set + Bi-temporal + member_status + Demo

**日期**：2026-04-28

### P0.5 可靠性修复（6 项）

| 修复 | 文件 | 说明 |
|------|------|------|
| `_build_author_map` 安全处理 | `extractor.py` | 跳过空 id、system/webhook/anonymous |
| 否定极性误报 | `store.py` | `_NEGATION_SAFE_WORDS` 豁免词（不管/不错/没问题等） |
| Ambiguous 后处理 | `extractor.py` | confidence<=0.3+ambiguous 丢弃 |
| 跨时区排序 | `extractor.py` | datetime 解析替代字符串排序 |
| Layer 3 字段保护 | `store.py` | owner/status 不同不 merge |
| 端到端集成测试 | `test_v15_improvements.py` | 4 条消息完整流程 |

### Golden Set 评测闭环
- 新增 `examples/golden_set.jsonl`：30 条样本，12 种场景
- 新增 `scripts/run_golden_eval.py`：支持 `--llm`、`--compare`
- 新增 `docs/evaluation_report.md`
- 当前指标：29/30 (96.7%)

### Bi-temporal 时间线
- `schema.py`：`MemoryItem` 增加 `valid_from/valid_to/recorded_at`
- `store.py`：supersede 时设 valid_to，`list_items(as_of=...)` 支持时间点查询
- 跨时区比较：`_compare_iso_time` + `_parse_iso_as_utc`

### Member Status
- `extractor.py`：新增 `_extract_member_status()`（请假/出差/习惯用/擅长）
- `handoff.py`：SECTION_TITLES 增加成员状态

### Demo 叙事
- 新增 `docs/demo_script.md`、`docs/judge_qna.md`
- 重写 `README.md`：Memory Engine ≠ Chat History/RAG + 对比表

### 测试指标：59 tests pass，Golden Set 29/30 (96.7%)
